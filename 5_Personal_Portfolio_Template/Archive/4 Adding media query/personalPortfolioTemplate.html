Project 1 : Technical Documentation

This project is a template for a technial documentation. It uses one main section with a main-doc id. Inside this section, it has as many other sections as required. The template I made has five sections inside the main-doc section. This then allows one to input information about a subject.

In the example given, I call the subject myApp. I then seperate the headings of the documentation to describe the features of myApp and how to use it, the advantages of it's features, it's target audience and it's areas for improvement.

Project 2 : Book Feedback Form

This project uses features such as checkboxes , a drop down menu with choices and a text box. It is designed to be a feedback form where the user can select their answers from the options provided. Some important details include the fact that the user can choose as many checkboxes but can only choose one of the radio buttons provided.

As a result, the feedback form can be customised to fit the objective of what information the developer wants to obtain. There is a submit button at the end. The submit button will only work if the name and email input fields are filled in correctly. For the email input field, the user must even ensure there is a '@' somewhere in the input as all emails require to have a @. This makes the form validated with suitable inputs before it is submitted.

Project 3 : Product Landing Page

This project splits the elements into three sections - a nav bar , a opt-in field form and a list of frequently asked questions. The nav bar has internal links which when clicked, re-direct the user to a designated part of the web page. For example, clicking on ''opt-in'' will send the user to part of the web page which has input fields.

Again, like the Book Feedback Form project, all of the inputs are checked for validation before they are sent. The checks include that the name and email fields have been filled in and that the email field as a '@' somewhere.
